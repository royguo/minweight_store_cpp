## minweight_store WAL-backed 主链路

- 新增 `src/minweight_store` 实际 library 实现，替代原 interface-only target。
- 新增 public headers：
  - `include/minweight_store/store.h`
  - `include/minweight_store/options.h`
  - `include/minweight_store/runtime.h`
  - `include/minweight_store/status.h`
- 实现 `Runtime` 注入边界和 `StdRuntime`：
  - `NewMutex`
  - `NewRWMutex`
  - `BlockingIO`
- 实现 WAL-backed store 主链路：
  - `Open` / `Close`
  - `Put` / `Get` / `Delete` / `Len`
  - `Scan` / `ScanRange`
  - `ReverseScan` / `ReverseScanRange`
  - `SeekGE` / `SeekLE`
- WAL 复用 Go 版二进制布局：
  - 4096 字节 header。
  - `MWWAL01\0` magic。
  - record header 13 字节。
  - `file_no << 30 | offset` position 编码。
  - IEEE CRC32 校验。
- 默认恢复语义为 point-in-time prefix replay：
  - replay 遇到第一个坏 record 时截断到最后一个合法 offset。
  - 不跳过中间坏 record 恢复后续 record。
- 当前启动路径从 WAL 重建 heap node index，尚未实现 manifest、mmap node checkpoint、SST 和 compaction。

## minweight_store generation snapshot checkpoint

- 实现 C++ 轻量 checkpoint：
  - `MANIFEST`：单记录、固定长度、CRC 校验，记录当前 WAL generation。
  - `SNAPSHOT.<generation>`：保存 checkpoint 时刻 live key/value，header 校验 generation，每条 record 带 CRC。
  - WAL 文件布局改为 `wal/<generation>/<file_no>.wal`。
  - checkpoint 后切换到下一代空 WAL，并清理旧 WAL generation / snapshot。
- 恢复路径：
  - 有合法 manifest 时，加载同 generation snapshot，再 replay 当前 generation WAL tail。
  - 无 manifest 时，从第 1 代 WAL 直接 replay。
  - snapshot 按 generation 命名，避免固定名 snapshot 在 manifest rename 前被覆盖导致新旧状态混读。
- 修复 Delete 返回值：
  - `Delete` 成功后不再把 OK `Status` 隐式构造成 `Result<bool>{false}`。
  - 删除路径现在先从 index 移除，再追加 tombstone；tombstone 失败时尝试恢复旧 position。
- 增加测试：
  - Close checkpoint 后重开。
  - checkpoint 后继续写入 WAL tail，重开时 snapshot + tail replay。
  - WAL point-in-time prefix recovery。
  - WAL strict recovery failure。
